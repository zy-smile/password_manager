# 本地密码管理 App 项目架构设计文档

## 一、文档说明

### 1.1 文档目的

本文档用于定义本地密码管理 App（Password Vault）第一阶段的整体技术架构、模块职责、代码组织方式、数据流转方式及安全体系设计，作为项目开发的核心技术指导文档。

目标：

- 保证项目可维护性
- 保证代码可扩展性
- 保证安全体系完整性
- 保证后续云同步版本可平滑升级

---

## 二、项目目标

### 第一阶段目标（V1.0）

实现一个：

## 本地离线 + 高安全 + 高可维护

的密码管理 App。

核心能力：

- 主密码系统
- 本地加密存储
- 生物识别解锁
- 账号密码管理
- 搜索 / 分类 / 收藏
- 自动锁屏
- 密码生成器

暂不实现：

- 云同步
- 多设备同步
- 企业协作
- 权限管理系统

## 六、各层职责说明

---

# 1. Presentation Layer

职责：

- 页面展示
- 用户交互
- 状态管理
- Provider 管理
- UI 组件封装

禁止：

- 直接操作数据库
- 直接写加密逻辑

原则：

UI 只负责展示，不负责业务。

---

# 2. Domain Layer

职责：

- 业务规则
- UseCase
- Entity
- Repository 抽象

特点：

最稳定的一层。

未来即使换数据库、换框架，也无需修改。

---

# 3. Data Layer

职责：

- Drift 数据库
- 本地数据源
- Repository 实现
- Model 转换

特点：

负责真正的数据读写。

---

# 4. Core Layer

职责：

- AES 加密
- 主密码派生
- Secure Storage
- 生物识别
- 工具函数

特点：

系统最底层能力。

---

## 七、数据流设计

---

## 示例：新增账号流程

```text
用户点击保存
→ UI Page
→ Provider
→ UseCase
→ Repository Interface
→ Repository Impl
→ Local DataSource
→ Drift Database
→ 返回结果
→ UI 刷新
```

原则：

## 单向数据流

便于维护和调试。

---

## 八、安全架构设计

---

# 核心原则

## 永远不保存明文密码

必须：

- 所有密码 AES 加密
- 主密码不可逆保存
- 密钥独立保护

---

# 主密码体系

```text
用户输入 Master Password
→ PBKDF2 派生密钥
→ 生成 AES Key
→ 加密账号密码
→ Salt 单独保存
→ Key 保存到 Secure Storage
```

说明：

- Android：Keystore
- iOS：Keychain

安全等级高。

---

# 自动锁屏机制

触发条件：

- App 进入后台
- 超过设定时间无操作
- 手动立即锁定

处理：

统一跳转 Unlock Page

---

# 生物识别机制

支持：

- 指纹
- Face ID

策略：

优先使用生物识别

失败后：

回退主密码验证

## 十、状态管理设计

---

## 使用 Riverpod

原因：

- 解耦优秀
- 可测试性强
- 生命周期管理清晰
- 比 Provider 更现代

推荐：

```text
StateNotifier + State
```

不建议：

```text
大量 setState
```

---

## Provider 示例

```text
VaultProvider
AuthProvider
SettingsProvider
BiometricProvider
LockProvider
```

职责清晰分离。

---

## 十一、异常处理机制

---

统一异常分类：

- 数据库异常
- 解密失败
- 生物识别失败
- 主密码验证失败
- 权限异常

统一处理：

- UI Toast
- 日志记录
- Fail Safe 回退机制

原则：

不能因为异常导致密码丢失。
